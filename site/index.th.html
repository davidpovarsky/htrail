เนทีฟ ไม่ใช่ Electron

Swift + SwiftUI ล้วน บนแกน SwiftNIO เปิดได้ทันที ใช้หน่วยความจำน้อย และให้ความรู้สึกเหมือนแอป Mac เพราะมันเป็นแอป Mac จริง ๆ

เป็นเรื่องในเครื่องโดยการออกแบบ

ทั้งพร็อกซี ผู้ออกใบรับรอง (CA) ทราฟฟิกที่ดักจับได้ และประวัติการส่งซ้ำ ล้วนอยู่บนเครื่องของคุณ ไม่มีเซิร์ฟเวอร์ HTTrail ให้ส่งข้อมูลใด ๆ ไปหา

เป็นของคุณที่จะวางใจได้

กุญแจส่วนตัวของ root CA ถูกสร้างขึ้นบนอุปกรณ์และไม่เคยออกจากเครื่อง คุณติดตั้งเอง วางใจเอง และถอนออกเอง — อยู่ในความควบคุมของคุณทั้งหมด

// ความสามารถ

ชุดเครื่องมือ ดีบักครบครัน ในหน้าต่างเดียว

สองผลิตภัณฑ์หลอมรวมกัน: พร็อกซีดักจับสไตล์ Charles และเครื่องมือ API สไตล์ Hoppscotch ใช้เอนจินดักจับร่วมกันชุดเดียว

พร็อกซี MITM สำหรับ HTTPS

ตอบ CONNECT ปิดการเชื่อมต่อ TLS ด้วย leaf cert ที่เพิ่งสร้างใหม่ ส่งต่อไปยัง origin จริง และดักจับทั้งสองทิศทาง — ทดสอบใช้งานจริงกับโฮสต์จริง

ผู้ออกใบรับรอง (CA) บนอุปกรณ์

สร้าง root แบบ self-signed P-256 (อายุ 10 ปี เก็บถาวร) และสร้าง leaf cert ต่อโฮสต์แบบทันที ติดตั้งและวางใจได้ในคลิกเดียวบน macOS

เอนจินกฎสไตล์ Charles

บล็อก, map local, map remote, เขียนคำขอหรือการตอบกลับใหม่, จำกัดแบนด์วิดท์ และตั้ง breakpoint แบบโต้ตอบ — หยุดทราฟฟิกไว้ แก้ไขเนื้อหา แล้วเดินหน้าต่อ

ตัวตรวจสอบทราฟฟิกเชิงลึก

เฮดเดอร์, เนื้อหา JSON ทั้งแบบดิบและจัดรูปสวยงาม, พรีวิว HTML สด, พรีวิวรูปภาพ พร้อมสถานะ เวลา และขนาดของทุกทราฟฟิกที่ดักจับได้

เครื่องมือเขียนคำขอ

เมท็อด, query params, เฮดเดอร์, ทุกโหมดของเนื้อหา, GraphQL และตัวแปร {{env}} — พร้อมตัวช่วยตรวจสอบสิทธิ์แบบ Bearer, Basic และ API-key ในตัว

ไคลเอนต์เรียลไทม์

WebSocket, Server-Sent Events (SSE), Socket.IO (handshake แบบ Engine.IO v4) และไคลเอนต์ MQTT 3.1.1 — ทั้งหมดบนแกน NIO เดียวกัน

นำเข้าและส่งออก

นำเข้าจาก cURL, OpenAPI 3 และ Postman v2.1 ส่งออกเป็น cURL, Swift, JavaScript หรือ Python — และส่งออกทราฟฟิกที่ดักจับได้เป็น HAR 1.2

สคริปต์ก่อนคำขอและสคริปต์ทดสอบ

รัน JavaScript ทั้งก่อนและหลังคำขอผ่าน JavaScriptCore ด้วย API pm.* ที่คุ้นเคยสำหรับการตรวจสอบและค่าแบบไดนามิก

รายการอนุญาตสำหรับ SSL-proxying

ถอดรหัสเฉพาะโฮสต์ที่คุณระบุไว้ ส่วนที่เหลือจะถูกอุโมงค์แบบ blind โดยไม่แตะต้อง เพื่อให้แอปที่ใช้ certificate pinning ยังทำงานควบคู่ไปกับการดักจับของคุณได้

จากศูนย์สู่ ทราฟฟิกที่ถอดรหัสแล้ว

1

เริ่มพร็อกซี

กดปุ่มเล่นบนแถบเครื่องมือหรือ ⌘P HTTrail จะรอรับการเชื่อมต่อที่ 127.0.0.1:9090

2

วางใจ CA

คลิกเดียวเพื่อติดตั้งและวางใจ root CA บน macOS — หรือติดตั้งโปรไฟล์ .mobileconfig บน iOS

3

ดักจับ

เปิดสวิตช์พร็อกซีของระบบ แล้วทุกคำขอและการตอบกลับจะไหลเข้าสู่แท็บ Capture แบบเรียลไทม์

4

เขียนคำขอและส่งซ้ำ

แก้ไขแล้วส่งทราฟฟิกใด ๆ ซ้ำ คัดลอกเป็น cURL หรือสร้างคำขอใหม่แล้วกด Send (⌘↵)

สร้างมาเพื่อ Mac และ iPhone

เอนจินดักจับ, CA และโมเดลแอปถูกแชร์ไว้ในแกน Swift เดียว ทำให้ทั้งสองแอปทำงานสอดคล้องกันเสมอ

macOS

ดีบักเกอร์เต็มรูปแบบ พร้อมการผสานรวมกับระบบแบบเนทีฟ

  • สลับพร็อกซีของระบบผ่าน networksetup
  • ติดตั้งและวางใจ CA เข้าสู่ System keychain ได้ในคลิกเดียว
  • เปิดดูและส่งออก root CA จาก Finder
  • ตัวตรวจสอบ, เครื่องมือเขียนคำขอ, กฎ และไคลเอนต์เรียลไทม์ครบถ้วน

iOS

ดักจับ iPhone เครื่องนี้ หรืออุปกรณ์อื่นผ่าน Mac ของคุณ

  • VPN ดักจับบนอุปกรณ์ผ่านส่วนขยาย Packet Tunnel
  • การดักจับยังทำงานต่อแม้แอปอยู่เบื้องหลัง
  • ติดตั้ง CA + พร็อกซีผ่าน .mobileconfig ไฟล์เดียว
  • หรือชี้พร็อกซีของอุปกรณ์ใด ๆ มาที่ Mac ของคุณ — ไม่ต้องมีบัญชีนักพัฒนา

ทราฟฟิกของคุณไม่เคยออกจากอุปกรณ์ของคุณ

HTTrail ไม่มีการวิเคราะห์ข้อมูล ไม่มีบัญชี และไม่มีแบ็กเอนด์ ข้อมูลที่มันถอดรหัสคือข้อมูลที่อ่อนไหวที่สุดที่คุณมี — โทเคน, คุกกี้, เนื้อหาคำขอ — จึงอยู่บนเครื่องของคุณทั้งหมด และเราออกแบบมาให้เราไม่มีทางได้รับมันเลย
